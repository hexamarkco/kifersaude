Visao geral

O que esta publicado hoje

O edge function leads-api expoe rotas para monitoramento, consulta, criacao, atualizacao e arquivamento de leads.

Regras de acesso

GET /leads e GET /leads/:idPermitido para perfis admin e observer.
POST, PUT, DELETE e batchRestrito a admin.
Rotas internasAlgumas acoes aceitam somente service role.

Filtros suportados

status, responsavel, origem, tipo_contratacaoAceitam id ou valor textual equivalente.
telefoneNormalizado para digitos no backend.
emailFiltro por ilike.

Regras de payload

Obrigatorios na criacaonome_completo, telefone, origem, tipo, status e responsavel.
Datas aceitasISO 8601 ou YYYY-MM-DD.
DuplicidadeLeads duplicados podem ser marcados automaticamente com status Duplicado.

Autenticacao

Headers esperados

A API usa token do Supabase no header Authorization. Para chamadas em navegador ou ferramentas HTTP, envie tambem apikey.

Headers basicos 
Authorization: Bearer SEU_TOKEN_SUPABASE
apikey: SEU_TOKEN_SUPABASE
Content-Type: application/json

Para healthcheck, o endpoint responde sem leitura de dados sensiveis, mas as rotas de leads validam o usuario autenticado e o papel associado ao perfil.
