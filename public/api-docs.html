Visao geral

O que esta publicado hoje

O edge function leads-api expoe rotas para monitoramento, consulta, criacao, atualizacao e arquivamento de leads.

Regras de acesso

GET /leadsPermitido para perfis admin e observer.
POST /leads, PUT /leads/:id e POST /leads/batchRestrito a admin.
Rotas internasAlgumas acoes aceitam somente service role.

Filtros suportados

status, responsavel, origem, tipo_contratacaoAceitam id ou valor textual equivalente.
telefoneNormalizado para digitos no backend.
emailFiltro por ilike.

Regras de payload

Obrigatorios na criacaonome_completo, telefone, origem, tipo, status e responsavel.
Datas aceitasISO 8601 ou YYYY-MM-DD.
DuplicidadeLeads duplicados podem ser marcados automaticamente com status Duplicado.

Autenticacao

Headers esperados

A API usa token do Supabase no header Authorization. Para chamadas em navegador ou ferramentas HTTP, envie tambem apikey.

Headers basicos 
Authorization: Bearer SEU_TOKEN_SUPABASE
apikey: SEU_TOKEN_SUPABASE
Content-Type: application/json

Para healthcheck, o endpoint responde sem leitura de dados sensiveis, mas as rotas de leads validam o usuario autenticado e o papel associado ao perfil.

Status codes

              
Respostas mais comuns

              
Os retornos abaixo resumem o comportamento observado no edge function atual.

            

          

          

            

              
Sucesso

              

                
200 OKHealthcheck, listagens, update, batch e rotas internas bem-sucedidas.

                
201 CreatedCriacao individual de lead.

              

            

            

              
Falhas

              

                
400 Bad RequestPayload invalido, filtros invalidos ou body fora do formato esperado.

                
401 UnauthorizedToken ausente, invalido ou rota interna sem credencial adequada.

                
403 ForbiddenUsuario autenticado sem permissao para a acao.

                
404 Not FoundEndpoint inexistente ou lead nao encontrado em fluxos internos com lead_id.

                
500 / 502Erros internos de banco ou integracoes de automacao.

              

            

          

          
Dica operacional: em erros 400, consulte o campo details do JSON para identificar a validacao que falhou.
